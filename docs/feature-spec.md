## 1. 媒体管理模块

### 1.1 导入媒体文件

**触发方式**：
- 菜单栏 `文件 → 导入视频/音频`
- 拖拽文件到应用窗口
- 启动页"打开文件"按钮
- 快捷键 `Ctrl/Cmd + O`

**支持的视频格式**：
| 容器 | 扩展名 | 视频编码 | 音频编码 |
|------|--------|----------|----------|
| MP4 | .mp4, .m4v | H.264, H.265 | AAC, MP3 |
| Matroska | .mkv | H.264, H.265, VP9 | AAC, MP3, Opus, FLAC |
| WebM | .webm | VP8, VP9, AV1 | Opus, Vorbis |
| AVI | .avi | H.264, MJPEG | MP3, PCM |
| MOV | .mov | H.264, ProRes | AAC, PCM |
| FLV | .flv | H.264 | AAC, MP3 |

**支持的音频格式**：
| 容器 | 扩展名 | 编码 |
|------|--------|------|
| MP3 | .mp3 | MPEG Audio Layer 3 |
| WAV | .wav | PCM, IEEE Float |
| FLAC | .flac | FLAC |
| AAC | .aac, .m4a | AAC |
| Ogg | .ogg | Vorbis, Opus |
| WMA | .wma | WMA |

**导入行为**：
1. 用户选择文件后，前端显示加载动画
2. 后端检测文件格式与编码：
   - 视频文件：提取视频流+音频流信息（分辨率、帧率、编码、时长、码率）
   - 纯音频文件：提取音频流信息（采样率、声道数、编码、时长、码率）
3. 后端生成音频波形数据（峰值数组，用于时间线显示）：
   - 采样精度：每像素2-4个采样点（根据缩放级别）
   - 数据格式：归一化到[-1, 1]的f32数组
4. 后端检查WebView能否直接播放此格式：
   - 能播放 → 使用原始文件路径
   - 不能播放 → 后台转码为H.264 MP4，缓存到应用数据目录的`transcode_cache/`下
5. 如果文件只有视频没有音频轨道（或反之），显示警告
6. 加载完成后，视频预览显示第一帧，时间线显示波形

**错误处理**：
- 文件不存在 → 错误提示，保留在最近文件列表但标记为"缺失"
- 文件损坏/不支持的编码 → 错误提示"无法解码此文件，请转换为常见格式后重试"
- 文件过大（>4GB）→ 警告"文件较大，可能影响性能，建议先压缩视频"
- 导入后系统存储不足 → 提示清理缓存

**关联导入**：
- 支持分别导入视频和音频（替换视频中的音频轨道）
- 导入音频后可以再导入视频（纯歌词视频制作）
- 支持仅导入音频（制作LRC歌词文件，无需视频）

### 1.2 媒体预览

**视频预览区**：
- 使用HTML5 `<video>` 元素播放
- 显示分辨率、帧率、当前时间码（`HH:MM:SS.ms`）
- 支持全屏预览（F11或双击）
- 画面缩放适配：`适合宽度` / `适合高度` / `100%` / `自由缩放`
- 安全区域指示线（可选显示）：标出4:3/16:9安全区、标题安全区

**音频波形图**：
- 显示完整音频波形（在时间线区域）
- 当前播放位置用红色竖线标记
- 已打轴区域用半透明色块覆盖
- 波形图可缩放（横向：时间精度；纵向：振幅）
- 鼠标悬停显示精确时间

**时间线面板**：
```
┌──────────────────────────────────────────────────┐
│ 00:00.000              01:23.456         03:45.789│  ← 时间刻度
│ ▓▓▓▓▓░░░░▓▓▓▓▓▓░░░░░▓▓▓▓▓▓▓░░░░░▓▓▓▓░░░░░░▓▓▓░│  ← 波形图
│ ═══════                      ═══════════         │  ← 歌词行时间条
│ ─┬──┬──┬──┬─               ─┬──┬──┬──┬──┬──    │  ← 逐字时间标记
└──────────────────────────────────────────────────┘
```

### 1.3 视频/音频分离

- 分离视频中的音频为独立音频轨道（供仅导出音频+歌词使用）
- 替换音频：移除当前音频，导入新音频文件
- 音量调整：音频增益控制（-20dB ~ +20dB）
- 音频淡入淡出：可设置开始/结束处的淡入淡出时长

## 2. 歌词编辑模块

### 2.1 文本输入

**手动输入**：
- 在歌词编辑面板直接输入/粘贴文本
- 每行一个换行（Enter）
- 支持Unicode全字符集（中文、日文、韩文、英文、特殊符号等）
- 实时字数统计

**文件导入**：
| 格式 | 扩展名 | 说明 |
|------|--------|------|
| 纯文本 | .txt | UTF-8，每行一句歌词 |
| LRC歌词 | .lrc | 标准LRC格式，提取文本和行级时间 |
| 增强LRC | .lrc | 支持逐字时间标签 `<00:01.23>` |
| ASS字幕 | .ass | 提取文本内容 |
| SRT字幕 | .srt | 提取文本内容 |

**LRC导入详细行为**：
- 识别标准行级标签 `[mm:ss.xx]`
- 识别逐字标签 `[mm:ss.xx]` （用于增强LRC）
- 解析出的时间自动填充到对应行或字
- 不支持的标签保留原文但标记为"未解析"
- 导入后提示"已导入X行，其中Y行包含时间信息"

**文本预处理**：
- 自动去除行首行尾空白
- 合并空行为单个空白行（保留段落分隔）
- 中英文之间自动插入空格（可选，默认开启）
- 全角符号转半角（可选，默认关闭）

### 2.2 文本分行规则

**自动分行**（用户粘贴一整段文本时）：
1. 按换行符分行（优先）
2. 按标点符号分行：句号、逗号、问号、感叹号等作为分行提示
3. 智能分行：检测到连续8-16个字符无标点时，在适当位置建议分行
4. 分行后每行字符数建议：中文8-16字/行，英文40-60字符/行

**手动分行**：
- 在任意位置按Enter分行
- 合并相邻行：退格键或专用合并按钮

### 2.3 分字/分词

**中文模式（逐字）**：
- 每个汉字、标点符号、英文单词作为一个独立时间单元
- 标点符号默认不单独标记，附加到前一个字（可配置）

**日文模式（逐字/逐词）**：
- 每个假名、汉字作为一个时间单元
- 可选择"按词分割"模式（对于复合词不拆开）

**英文/拼音文字模式（逐词/逐音节）**：
- 默认按单词分割
- 可选按音节分割

**手动调整分词**：
- 合并：选择两个字→右键→合并
- 拆分：选择组合→右键→拆分
- 拖拽分界线调整分词边界

### 2.4 歌词编辑操作

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 插入文字 | 直接输入 | 在光标位置插入 |
| 删除文字 | Backspace/Delete | 删除光标前/后文字 |
| 替换文字 | 选中后输入 | 替换选中文本 |
| 移动行 | Ctrl+↑/↓ | 上下移动整行 |
| 复制行 | Ctrl+D | 复制当前行 |
| 删除行 | Ctrl+Shift+K | 删除整行 |
| 撤销 | Ctrl+Z | 撤销操作 |
| 重做 | Ctrl+Shift+Z | 重做操作 |
| 查找替换 | Ctrl+F | 打开查找替换面板 |

## 3. 打轴模块

打轴（Timing）是整个软件的核心功能。用户通过按特定按键，在音频播放过程中标记每个字/词的起止时间点。

### 3.1 打轴工作流

```
准备阶段:
  1. 确保媒体已加载、歌词已输入
  2. 检查分字结果是否满意（必要时手动调整）
  3. 切换到「打轴模式」(快捷键: F2 或工具栏按钮)

录制阶段:
  1. 按 Space（或自定义打轴键）开始播放
  2. 音频播放中，每按一次打轴键 → 标记当前时间，游标前进到下一个字
  3. 按 Esc 或再次按打轴键 → 暂停播放
  4. 暂停后可后退微调，然后继续

完成阶段:
  1. 所有字都标记完成后，自动停止
  2. 检查时间线，手动微调不准确的标记
  3. 逐行预览扫色效果
```

### 3.2 打轴模式详解

**模式A：实时打轴（默认推荐）**

```
状态机:
  IDLE ──[打轴键按下]──→ PLAYING_AND_MARKING
  PLAYING_AND_MARKING ──[打轴键按下]──→ 记录时间点 + 游标前进
  PLAYING_AND_MARKING ──[暂停键(Esc)]──→ PAUSED
  PAUSED ──[打轴键按下]──→ PLAYING_AND_MARKING
  PAUSED ──[退格键]──→ 撤销最后标记 + 游标后退 (可连续)
  PAUSED ──[←/→]──→ 微调当前时间位置
  PAUSED ──[播放键(Space)]──→ PLAYING (不打轴)
  IDLE ──[播放键]──→ PLAYING (不打轴)
  
  当所有字符都已标记:
  PLAYING_AND_MARKING → AUTO_PAUSE_AND_NOTIFY
```

**模式B：Tap打轴（高级模式）**

在此模式下，用户无需跟随播放，而是：
1. 音频不播放
2. 用户根据节奏感匀速按打轴键
3. 系统根据按键间隔自动分配时间
4. 适合节奏感强的用户快速打轴

完成后系统播放一遍供验证。

**模式C：打节拍点模式**

1. 用户仅标记每句的起始时间和结束时间（2个按键/行）
2. 句内各字的时间均匀分配
3. 然后再微调个别字

### 3.3 打轴键行为

**主打轴键（默认Space）**：
- 短按（<200ms）：标记当前字开始 + 游标前进 + 继续播放
- 长按（≥200ms）：标记当前字开始 + 游标前进 + 暂停（方便精确控制）

**辅助操作键**：
| 按键 | 功能 |
|------|------|
| Space | 主打轴键（播放中）/ 播放/暂停（非打轴模式） |
| Esc | 暂停播放 + 退出打轴 |
| Backspace | 暂停时：撤销上一个标记 + 游标后退 |
| ← | 后退100ms |
| → | 前进100ms |
| Shift+← | 后退1秒 |
| Shift+→ | 前进1秒 |
| ↑ | 跳转到上一行开始 |
| ↓ | 跳转到下一行开始 |
| Tab | 标记当前字结束并跳到下一行（用于行末快速操作） |
| 数字键1-9 | 标记当前字 + 设置特殊属性（如：1=正常，2=转音，3=换气） |

### 3.4 提前量（Lead-in）

打轴时用户反应有延迟，需要设置提前量：

- **全局提前量**：用户设置（如-50ms），所有标记自动减去此值
- **可在时间线中实时调整**
- **推荐值**：初学者-80ms，熟练者-30ms，专业-10ms

### 3.5 自动吸附功能

标记时间点自动微调到最近的参考点：

- **波形峰值吸附**（默认开启）：吸附到±30ms内的波形峰值
- **节拍吸附**：如果用户设置了BPM，吸附到最近的节拍点
- **吸附强度**可调：无 / 弱(±15ms) / 中(±30ms) / 强(±50ms)

### 3.6 打轴辅助功能

**预卷（Pre-roll）**：
- 从标记点前X秒开始播放（X可配置，默认2秒）
- 给用户提供节奏上下文

**节拍器（Metronome）**：
- 设置BPM（手动输入或用Tap Tempo）
- 打轴时播放节拍音（可关闭）
- 时间线上显示节拍网格

**标记提示**：
- 当前待标记的字符高亮闪烁
- 已标记字符显示为完成色
- 即将到来的字符显示半透明预览

**速度修正**：
- 如果播放速度过快导致打轴跟不上，可以降低播放速度：
  - 0.5x / 0.75x / 0.9x / 1.0x（正常）/ 1.25x / 1.5x
- 降速打轴的时间戳自动换算为正常速度

### 3.7 时间微调

**时间线拖拽**：
- 每个字在时间线上显示为可拖拽的分隔线
- 拖拽时间线标记左右移动
- 按住Shift拖拽：移动当前及之后所有标记（连锁移动）
- 按住Ctrl拖拽：微调模式（放大100倍精度）

**波形辅助微调**：
- 放大波形图观察音频细节
- 标记线应与音频能量起始点对齐
- 特定波形特征（如辅音、元音）可用不同颜色标注

**批量调整**：
- 选择多个时间点 → 整体偏移
- 选中一段 → 拉伸/压缩时间
- 选中一行 → 均匀重新分配时间

**数值输入**：
- 双击时间标记 → 输入精确毫秒值
- 在属性面板中编辑开始时间/结束时间/持续时长

## 4. 扫色渲染模块

### 4.1 扫色动画效果

**核心效果：双色逐字填充**

每个字根据当前播放时间计算填充比例：

```
fillRatio = clamp((currentTime - charStartTime) / (charEndTime - charStartTime), 0, 1)
```

填充方向（Fill Direction）：
| 模式 | 描述 | 适用场景 |
|------|------|----------|
| `left-to-right` | 从左到右水平填充 | 横排中文、英文 |
| `right-to-left` | 从右到左水平填充 | 阿拉伯语 |
| `top-to-bottom` | 从上到下垂直填充 | 竖排文字 |
| `bottom-to-top` | 从下到上垂直填充 | 特殊效果 |

**渐变过渡带**：
- 在填充边界有一个可配置宽度的渐变过渡（1-10px）
- 过渡带类型：线性 / 缓入 / 缓出 / 平滑阶梯

**逐字动画曲线**：
- 每个字独立应用缓动曲线（easing function）
- 预设：`linear`, `ease-in`, `ease-out`, `ease-in-out`
- 默认：`linear`（与演唱同步最准确）

### 4.2 文字样式

**基础属性**：
| 属性 | 可选值 | 默认值 |
|------|--------|--------|
| 字体 | 系统字体列表 + 自定义字体 | Sans-serif |
| 字号 | 12-200px | 48px |
| 未唱颜色 | 任意RGBA | `rgba(255,255,255,1)` |
| 已唱颜色 | 任意RGBA | `rgba(255,60,60,1)` |
| 字间距 | 0-50px | 2px |
| 行间距 | 0-100px | 12px |
| 粗体 | on/off | off |
| 斜体 | on/off | off |

**描边/轮廓（Outline）**：
| 属性 | 可选值 | 默认值 |
|------|--------|--------|
| 描边宽度 | 0-20px | 2px |
| 描边颜色 | 任意RGBA | `rgba(0,0,0,1)` |
| 描边样式 | 外描边/内描边/居中 | 外描边 |

**阴影（Shadow）**：
| 属性 | 可选值 | 默认值 |
|------|--------|--------|
| 阴影偏移X | -20 ~ 20px | 2px |
| 阴影偏移Y | -20 ~ 20px | 2px |
| 阴影模糊 | 0-30px | 4px |
| 阴影颜色 | 任意RGBA | `rgba(0,0,0,0.5)` |

**发光（Glow）**：
| 属性 | 可选值 | 默认值 |
|------|--------|--------|
| 发光半径 | 0-50px | 0（禁用） |
| 发光颜色 | 任意RGBA | `rgba(255,200,0,0.8)` |
| 发光强度 | 0-100% | 80% |

**背景条（Background Bar）**：
| 属性 | 可选值 | 默认值 |
|------|--------|--------|
| 背景条 | 开启/关闭 | 开启 |
| 背景颜色 | 任意RGBA | `rgba(0,0,0,0.5)` |
| 背景条高度 | 固定/自适应 | 自适应+padding |
| 背景条圆角 | 0-20px | 8px |
| 宽度模式 | 文字宽度/全屏宽度/固定宽度 | 文字宽度 |

### 4.3 歌词排版布局

**行排列**：
- 水平居中 / 左对齐 / 右对齐
- 垂直位置：顶部 / 居中 / 底部 / 自定义Y坐标（%）
- 双行模式：当前行在上、下一行在下（或反过来）

**竖排模式**：
- 从右到左排列
- 文字从上到下
- 扫色方向变为 top-to-bottom

**安全区适配**：
- 显示/隐藏标题安全区/动作安全区参考线
- 歌词自动避开安全区边界
- 预览时叠加半透明安全区指示

### 4.4 Canvas渲染管线

```
每帧 (requestAnimationFrame):
  1. 获取当前播放时间 currentTime
  2. 确定当前显示的行范围 (visibleLines)
  3. 对于每行:
     a. 计算整行在Canvas上的位置
     b. 绘制背景条 (如果启用)
     c. 对于行内每个字:
        - 计算 fillRatio = getFillRatio(char, currentTime)
        - 绘制未唱颜色部分 (通过clip控制可见区域)
        - 绘制已唱颜色部分 (fillRatio控制clip宽度)
        - 绘制描边
        - 绘制阴影
        - 绘制发光
     d. 绘制注音文字 (如果有)
  4. 绘制安全区参考线 (如果启用)
```

**性能优化**：
- 脏矩形检测：只重绘时间有变化的行
- 离屏Canvas缓存：静态元素（安全线、背景等）缓存到离屏Canvas
- 文字度量缓存：避免每帧重复调用`measureText()`
- 当播放停止时暂停渲染循环

## 5. 注音模块

### 5.1 注音模式

| 模式 | 适用语言 | 示例 |
|------|----------|------|
| 拼音（Pinyin） | 中文 | 我 → wǒ |
| 假名（Furigana） | 日文 | 私 → わたし |
| 罗马字（Romaji） | 日文 | 私 → watashi |
| 注音符号（Bopomofo） | 中文（台湾） | 我 → ㄨㄛˇ |
| 谚文（Hangul） | 韩文汉字 | 愛 → 애 |
| 自定义 | 任意 | 用户自定义注音文本 |

### 5.2 自动注音

**中文拼音**：
- 基于内置词典（CC-CEDICT + pinyin词典）查询
- 支持多音字常用读音选择
- 支持声调标记（默认带声调，可选无声调数字标记如"wo3"）

**日文假名**：
- 基于JMDict + kanjidic2词典查询
- 支持复合词读音
- 送假名处理（如：食べる → たべる，标记为 食{た}べる）

**批量注音**：
- 全部歌词一键注音
- 仅对选中行注音
- 新输入文字实时注音（可选）

### 5.3 注音显示

**排版规则**：
```
   wǒ    ài    nǐ       ← 拼音 (上方，小字号)
   我    爱    你        ← 原文字 (下方，正常字号)

   わたし             ← 假名 (上方，小字号)
   私                 ← 原文字 (下方，正常字号)
```

- 注音文字字号 = 原文字字号 × 0.4 ~ 0.5（可配置）
- 注音与原文间距 = 原文字字号 × 0.1（可配置）
- 注音颜色可独立设置
- 注音也参与扫色动画（可选）

### 5.4 手动编辑注音

- 点击某个字的注音 → 进入编辑模式
- 直接输入修改
- 右键 → 清除注音 / 设为默认读音 / 从其他行同步

### 5.5 注音导入导出

- 导入带注音的文本格式：
  ```
  # format: hanzi{pinyin}
  我{wo3} 爱{ai4} 你{ni3}
  ```
- 导出时可选是否包含注音数据

## 8. 项目工程模块

### 8.1 新建项目

```
新建项目对话框:
  - 项目名称
  - 媒体文件选择 (可选，稍后导入)
  - 预设样式选择
  - 画布分辨率:
      - 跟随视频 (如果有视频)
      - 自定义: 1920×1080, 1280×720, 3840×2160, ...
  - 帧率:
      - 跟随视频 (如果有视频)
      - 自定义: 23.976, 24, 25, 29.97, 30, 50, 59.94, 60
```

### 8.2 保存项目

**保存（Ctrl+S）**：
1. 前端序列化所有歌词数据、样式、打轴数据为JSON
2. 后端将JSON写入project.json
3. 后端将project.json + media_references.json打包为ZIP
4. 可选嵌入字体文件
5. 写入.klproj文件

**另存为（Ctrl+Shift+S）**：
- 选择新路径
- 可选择是否复制媒体文件到项目目录

**自动保存**：
- 默认每5分钟自动保存
- 自动保存文件命名为 `[项目名].autosave.klproj`
- 异常退出后下次启动检测并提示恢复

### 8.3 打开项目

- 菜单栏 `文件 → 打开项目`
- 启动页最近项目列表
- 双击.klproj文件关联打开
- 拖拽.klproj到窗口

**加载流程**：
1. 解压ZIP读取project.json
2. 验证JSON结构与字段
3. 检查媒体文件路径是否有效
4. 路径无效 → 提示"媒体文件缺失: [路径]，请手动定位"
5. 加载字体（嵌入字体或系统字体）
6. 生成波形数据（如果媒体文件有效）
7. 恢复上次编辑状态（选中行、滚动位置）

### 8.4 最近项目

- 显示最近10个项目
- 每个项目显示：名称、路径、最后修改时间、缩略图
- 右键：从列表移除 / 在文件管理器中显示
- 项目不存在时显示"文件已移动或删除"

### 8.5 项目模板

- 保存当前项目（不含媒体）为模板
- 模板含：样式预设、歌词布局、导出设置
- 新建项目时可从模板创建

## 9. 导出模块

### 9.1 导出成品视频

**输出格式**：
| 格式 | 容器 | 视频编码 | 用途 |
|------|------|----------|------|
| MP4 (H.264) | .mp4 | libx264 | 通用兼容 |
| MP4 (H.265) | .mp4 | libx265 | 高压缩比 |
| WebM | .webm | libvpx-vp9 | Web使用 |
| MOV (ProRes) | .mov | prores_ks | 专业后期 |

**导出设置**：
| 设置 | 说明 | 默认值 |
|------|------|--------|
| 分辨率 | 输出分辨率 | 跟随视频/1920×1080 |
| 帧率 | 输出帧率 | 跟随视频/30fps |
| 视频码率 | Mbps | 8 Mbps (1080p) |
| 音频码率 | kbps | 192 kbps |
| 编码速度预设 | ultrafast ~ veryslow | medium |
| 音频编码 | AAC/MP3/Opus/PCM | AAC |

**歌词渲染到视频**：
- 方式A：FFmpeg drawtext滤镜（简单效果，快速）
- 方式B：逐帧Canvas渲染 → 帧序列 → FFmpeg合成（所有效果支持，较慢）
- 方式C：FFmpeg filter_complex自定义滤镜链（折中方案）

推荐：默认使用方式A快速导出预览版，方式B用于最终导出高质量版。

**导出进度**：
- 显示进度条（帧数/总帧数）
- 预估剩余时间
- 支持取消导出（已生成部分不保留）

### 9.2 导出LRC歌词文件

**标准LRC格式**：
```
[ti:歌曲名]
[ar:歌手名]
[al:专辑名]
[by:制作者]
[offset:0]
[00:12.34]第一句歌词
[00:15.67]第二句歌词
[00:18.90]第三句歌词
```

**增强LRC（逐字时间）**：
```
[00:12.34]歌<00:12.34>詞<00:12.67>製<00:13.01>作<00:13.34>
```

设置选项：
- 是否包含元数据（ti/ar/al/by）
- 时间格式：`[mm:ss.xx]` 或 `[mm:ss.xxx]`
- 编码：UTF-8（默认）/ UTF-16 / GBK

### 9.3 导出逐字时间戳JSON

**格式A：完整结构**
```json
{
  "version": "1.0",
  "metadata": {
    "title": "歌曲名",
    "artist": "歌手名",
    "createdBy": "Karaoke Lyrics Maker",
    "createdAt": "2024-01-01T00:00:00Z"
  },
  "lines": [
    {
      "index": 0,
      "text": "这是一句歌词",
      "startTime": 12340,
      "endTime": 15670,
      "characters": [
        {"char": "这", "startTime": 12340, "endTime": 12670, "pinyin": "zhè"},
        {"char": "是", "startTime": 12670, "endTime": 12990, "pinyin": "shì"},
        ...
      ]
    }
  ]
}
```

**格式B：简化格式（仅逐字）**
```json
[
  {"c": "这", "s": 12340, "e": 12670, "p": "zhè"},
  {"c": "是", "s": 12670, "e": 12990, "p": "shì"},
  ...
]
```

设置选项：
- 包含/排除注音数据
- 包含/排除行级时间
- 包含/排除元数据
- 格式化输出（缩进）或紧凑输出

### 9.4 导出ASS字幕

```
[Script Info]
Title: 歌曲名
ScriptType: v4.00+
PlayResX: 1920
PlayResY: 1080

[V4+ Styles]
Format: Name, Fontname, Fontsize, PrimaryColour, ...
Style: Default,Arial,48,&H00FFFFFF,&H000000FF,...

[Events]
Format: Layer, Start, End, Style, Name, MarginL, MarginR, MarginV, Effect, Text
Dialogue: 0,0:00:12.34,0:00:15.67,Default,,0,0,0,,{\kf$1230}歌{\kf$1340}词
```

### 9.5 批量导出

- 一次导出多种格式
- 导出预设：保存导出设置组合
- 批量导出多个项目

### 9.6 导出范围

- 全部导出（默认）
- 导出选中行
- 导出时间范围（如仅导出00:30-01:30段落）
